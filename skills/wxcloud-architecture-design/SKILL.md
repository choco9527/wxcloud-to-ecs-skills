# 目标架构设计

## 目标

基于审计结果，设计从云开发迁移到自建服务的目标架构，并明确各平台能力的替换关系。

这个 skill 解决的是“当前依赖的云开发能力，迁到自建架构后分别落在哪里、如何承载、如何切流”。

## 何时使用

当用户出现以下意图时，优先使用本 skill：

- 先出目标架构
- 先设计迁移方案
- 先规划 ECS 或其他服务器承载方式
- 先决定数据库、文件、任务、鉴权应该怎么落位
- 先确定灰度和回滚策略

如果用户还没有完成使用面盘点，应先使用 `wxcloud-project-audit`。

## 设计范围

至少覆盖以下七类能力映射：

1. 云函数 -> 常规后端服务
- HTTP API
- 内部任务
- 后台 worker
- 管理端操作入口

2. 云数据库 -> 自建数据层
- 关系型数据库
- 文档型数据库
- 兼容层与历史字段策略

3. 云存储 -> 文件服务或对象存储
- 上传入口
- 公网访问 URL
- 历史 fileID 兼容策略
- 聚合文件或缓存文件的落位

4. 云触发器 -> 自建任务调度
- cron
- systemd timer
- 队列或 worker
- 手动触发入口

5. 微信登录态 -> 自建鉴权
- code2session
- token/session
- 用户身份映射
- 本地开发调试凭证

6. 前端调用链路 -> 统一请求层
- API baseURL
- 环境切换
- 旧请求契约兼容
- 文件 URL 访问策略

7. 部署与发布链路
- 服务运行形态
- 反向代理
- HTTPS
- 健康检查
- 灰度与回滚

## 设计步骤

1. 从审计结果中抽出平台能力清单，不要直接从业务页面开始设计。
2. 为每类平台能力选择目标承载方式，并说明替换原因。
3. 设计最小可用部署拓扑，明确 API、数据库、文件、任务分别放在哪里。
4. 设计环境划分：本地、测试/体验、正式。
5. 设计迁移期兼容策略：请求契约、历史字段、文件地址、开发登录。
6. 设计灰度切流与回滚策略，确保旧链路可以短期保留。

## ECS 最小可用承载建议

当目标环境是 ECS 或其他独立服务器时，至少要明确：

### 1. 服务层
- 应用如何运行：进程、systemd、容器
- 是否需要反向代理
- 是否需要健康检查接口

### 2. 数据层
- 数据库选型
- 是否与应用同机
- 是否只监听内网或本机
- 本地调试如何接入远端数据层

### 3. 文件层
- 上传文件存放位置
- 是否走本机文件系统或对象存储
- 对外访问 URL 的生成方式

### 4. 任务层
- 定时任务如何触发
- 手动任务如何触发
- 任务日志如何记录
- 失败如何重试

### 5. 网络与发布
- 域名与 HTTPS 前置条件
- 小程序合法域名要求
- 灰度切流的入口位置
- 回滚时如何快速切回旧链路

这里关注的是迁移视角下的落位方案，不展开具体厂商控制台操作细节。

## 常见设计错误

设计阶段重点避免以下问题：

- 只设计 API，不设计文件、任务和鉴权链路
- 本地调试方案直接混入正式环境
- 数据库迁移方案没有考虑历史兼容字段或 raw 数据
- 文件访问只考虑上传，不考虑历史资源与聚合文件
- 定时任务只考虑自动触发，不考虑手动运维入口
- 没有为前端保留统一请求层和环境切换收口点
- 没有明确灰度范围和回滚路径

## 输出模板

输出时统一使用以下结构：

### 1. 当前架构摘要
- 当前云开发能力分布
- 当前高耦合对象
- 当前环境与发布方式摘要

### 2. 目标架构摘要
- 目标后端形态
- 目标数据层形态
- 目标文件层形态
- 目标任务层形态
- 目标前端接入形态

### 3. 平台能力替换表
按能力维度给出：
- 当前依赖
- 目标承载
- 迁移期兼容策略
- 风险说明

### 4. 环境划分策略
至少区分：
- 本地开发环境
- 测试或体验环境
- 正式环境

并说明每个环境的：
- API 入口
- 鉴权策略
- 文件访问策略
- 数据访问策略

### 5. ECS 落位建议
- 应用如何运行
- 数据库如何落位
- 文件如何落位
- 任务如何落位
- 反向代理与 HTTPS 如何承担入口职责

### 6. 灰度切流策略
- 切流前置条件
- 切流顺序
- 观察项
- 失败回退条件

### 7. 回滚策略
- 前端如何切回旧链路
- 后端如何回退版本
- 数据与文件链路有哪些不可逆风险

## 注意事项

- 设计时优先保证前端调用方式收口。
- 不要把本地调试策略直接带进正式环境。
- 对文件访问、后台任务和鉴权链路要单独设计，不要只画 API 层。
- 设计输出必须能直接作为后续函数迁移和前端切流的输入，而不是停留在概念层。
