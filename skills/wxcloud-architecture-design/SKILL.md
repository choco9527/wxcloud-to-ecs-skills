# 目标架构设计

## 目标

基于审计结果，设计从云开发迁移到自建服务的目标架构，并明确各平台能力的替换关系。

## 设计范围

- 云函数到后端服务/API 的映射
- 云数据库到自建数据库的映射
- 云存储到文件服务或对象存储的映射
- 云触发器到任务调度器的映射
- 微信登录态到自建鉴权的映射
- 前端请求层的切换策略

## ECS 承载建议

当目标环境是 ECS 或其他独立服务器时，至少要明确：

- 服务运行形态：进程、systemd、容器
- 反向代理与 HTTPS 入口
- 数据库与文件目录的落位
- 定时任务的承载方式
- 本地调试与正式环境的隔离方式

这里关注的是迁移视角下的落位方案，不展开厂商控制台操作细节。

## 输出要求

输出时统一包含：

- 当前架构摘要
- 目标架构摘要
- 平台能力替换表
- 环境划分策略
- 灰度切流策略
- 回滚策略

## 注意事项

- 设计时优先保证前端调用方式收口。
- 不要把本地调试策略直接带进正式环境。
- 对文件访问、后台任务和鉴权链路要单独设计，不要只画 API 层。
