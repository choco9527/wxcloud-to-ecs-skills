# 云函数迁移

## 目标

将云函数迁移到常规后端运行环境中，同时尽量降低前端改动范围。

## 迁移原则

- 优先迁移主链路函数
- 优先保持调用契约稳定
- 先做兼容，再逐步收口
- 本地调试与正式鉴权分开处理

## 工作流

1. 找到函数调用入口与所有调用方。
2. 明确该函数依赖的平台能力：数据库、文件、环境变量、身份信息、定时触发等。
3. 设计目标接口形态：HTTP API、内部任务、后台 worker 或混合形式。
4. 保留必要的旧入参和旧返回兼容层。
5. 明确需要迁移的数据访问层和权限判断。
6. 给出最小验证方法与回滚点。

## 输出要求

输出时统一包含：

- 当前函数职责
- 调用来源
- 平台耦合点
- 目标后端形态
- 兼容策略
- 验证方法
- 回滚点

## 注意事项

- 不要只改后端实现，必须同时核对前端或调用侧的取值假设。
- 不要把迁移期兼容壳当作长期最终结构。
- 如果一个函数同时承载多种职责，应先拆责任，再设计迁移顺序。
