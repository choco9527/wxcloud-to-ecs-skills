# 前端切流

## 目标

在后端替代能力已具备的前提下，将前端从云开发调用迁移到自建 API，同时保留清晰的调试、灰度与回滚路径。

## 关注点

- 请求层统一入口
- 环境判断逻辑
- 登录态切换
- 文件 URL 与资源访问
- 旧返回结构兼容
- 灰度范围与回滚方式

## 工作流

1. 收口所有前端请求入口。
2. 将环境判断逻辑集中到单处配置。
3. 替换云函数调用为统一 API 请求方式。
4. 识别文件 URL、聚合文件、缓存文件等资源访问链路。
5. 处理历史返回结构或字段结构兼容。
6. 制定联调清单、灰度策略和回滚方案。

## 输出要求

输出时统一包含：

- 当前调用方式摘要
- 切流入口
- 环境策略
- 资源 URL 策略
- 联调清单
- 灰度方案
- 回滚方案

## 注意事项

- 不要把开发环境判断和审核/正式环境判断混杂在多个文件里。
- 不要忽略上传、资源下载、聚合文件这些非普通 JSON 请求。
- 前端切流前必须先确认后端健康检查、鉴权和资源访问链路可用。
